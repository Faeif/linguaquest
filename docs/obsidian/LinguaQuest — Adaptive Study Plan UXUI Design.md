## Core Mental Model

```
ไม่ใช่ "เรียนบทที่ 1, 2, 3"
แต่คือ "เตรียมพร้อมสำหรับทริปจีน"

User ไม่ได้คิดว่า:
"วันนี้ฉันจะเรียน Lesson 3 เรื่อง Measure Words"

User คิดว่า:
"วันนี้ฉันจะฝึกสั่งอาหารในร้านก๋วยเตี๋ยว
 เพราะเดือนหน้าจะไปเที่ยวเซี่ยงไฮ้"

ทุก UX decision ต้องตอบคำถามนี้:
"User รู้มั้ยว่าถ้าทำอันนี้เสร็จ ชีวิตจริงจะดีขึ้นยังไง?"
```

## Topic Design — โครงสร้าง

```
Topic คือ "สถานการณ์ที่จะเจอในชีวิตจริง"

ไม่ใช่:
❌ "บทที่ 3: คำกริยา"
❌ "Grammar: Aspect Markers"

แต่คือ:
✅ "สั่งอาหารในร้านที่เมนูมีแต่ภาษาจีน"
✅ "บอก Taxi ว่าจะไปที่ไหน"
✅ "ขอเข้าห้องน้ำในร้านอาหาร"
✅ "ต่อรองราคาในตลาดนัด"

แต่ละ Topic มี:
- Title ที่เป็น situation จริง
- Description: "เมื่อทำสำเร็จแล้ว คุณจะทำอะไรได้"
- 2-4 Lessons
- Estimated time
- Skills ที่จะได้ (Speaking / Vocab / Grammar)
```

## Lesson Types

```
Type 1: AI Companion Session (หลัก)
────────────────────────────────────
→ บทสนทนา situational กับ AI
→ มี flow ชัด: เริ่ม → กลาง → จบ
→ Plot twist ตรงกลาง
→ เวลา 8-15 นาที

Type 2: Vocab Drill (Flashcard SRS)
────────────────────────────────────
→ คำศัพท์ที่จำเป็นสำหรับ topic
→ Speaking mode + Classic mode
→ เวลา 5-10 นาที

Type 3: Grammar Drill (AI Generated)
────────────────────────────────────
→ AI generate คำถาม fill-in-blank
→ ตามจุดอ่อนของ user โดยเฉพาะ
→ อธิบายภาษาไทย
→ เวลา 5-8 นาที

Type 4: Real Talk Challenge (Advanced)
────────────────────────────────────
→ ปลดล็อคหลังผ่าน Lessons ปกติแล้ว
→ Same topic แต่ no hints, native speed
→ เหมาะสำหรับ test ตัวเอง
```

## Lesson Output — User ได้อะไรกลับไป

```
หลังจบแต่ละ Lesson ผู้ใช้ได้:

1. Speaking Score + Tone Analysis
   Overall: 81%
   แต่ละตัวอักษร: 你🟢 好🟢 在🔴 哪🟡 里🟢

2. ประโยคหลักที่ต้องจำ (Key Phrases)
   "请问...在哪里？" — ขอถามว่า...อยู่ที่ไหน
   "一直走，然后左转" — เดินตรงไป แล้วเลี้ยวซ้าย
   "大概多远？" — ไกลประมาณไหน
   [📋 Copy ทั้งหมด] [💾 บันทึก Flashcard]

3. Grammar ที่ต้องจำ (จาก Pill ที่โชว์ระหว่าง session)
   "怎么走 = วิธีไป (ถามทาง)"
   "在 + place = อยู่ที่..."

4. Vocab ใหม่ที่เจอ
   左 右 前 直走 转 大概
   [+ เพิ่มใน SRS ทั้งหมด]

5. Badge ถ้าทำได้ดี
   "🎯 First Try! — ผ่านครั้งแรกโดยไม่ hint"
```

## Session → Study Plan Feedback Loop

```
หลังจบ AI Companion Session:

Speaking score < 60%:
  → ทำซ้ำ topic เดิม ใน scenario ใหม่
  → ลด difficulty session ถัดไป

Speaking score 60-80%:
  → ผ่าน, ไปต่อ lesson ถัดไป
  → Schedule tone drill ในอีก 2 วัน (spaced)

Speaking score > 80%:
  → Lesson marked done
  → Unlock ถัดไป
  → ถ้า > 90% → Real Talk challenge unlock

Grammar Pill โชว์ 3 ครั้ง (error ซ้ำ):
  → เพิ่ม Grammar Drill เข้าแผนอัตโนมัติ
  → โชว์ใน Day View พร้อม reason

Vocab Tap Popup กด > 5 คำต่อ session:
  → แสดงว่า vocab ไม่พอ
  → เพิ่ม Vocab Drill ก่อน Companion session ถัดไป

Hint ใช้ทุก turn:
  → Session ถัดไปยังอยู่ Learner mode
  → ไม่ขึ้น Real Talk จนกว่า hint < 2 ครั้ง/session
```

## Key UX Principles

```
1. ทุก Lesson บอก "จะได้อะไร" ก่อนเริ่ม
   ไม่ใช่ "Lesson 3: Grammar Practice"
   แต่คือ "หลังนี้คุณจะถามทางใน Uber ได้"

2. Progress = Real-world Readiness ไม่ใช่ % complete
   "62% พร้อมสำหรับทริปจีน"
   ไม่ใช่ "62% เสร็จแล้ว"

3. Regenerate ได้เสมอ แต่ต้องมีเหตุผล
   ไม่ใช่แค่กด refresh
   User เลือกว่าอยากได้อะไรแทน

4. AI changes = transparent ไม่ใช่ invisible
   บอก user ว่า AI ปรับอะไร และทำไม
   สร้าง trust ไม่ใช่ magic black box

5. Session จบ → ได้ phrasebook ส่วนตัว
   ประโยคทุกประโยคที่ฝึก → เก็บใน Progress
   ใช้ได้จริงระหว่างเดินทาง
```
